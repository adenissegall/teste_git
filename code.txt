cria uma pasta
git bash here
vai abrir o cmd do git
git init
cria uma arquivo e coloca na pasta
no bash do git, "git status" para ver o que já foi comitado dos arquivos da pasta
vai mostrar todos os arquivos que não foram preparados para o versionamento
para incluiir no versionsamento "git add" + o nome do arquivo, ex: git add "code.txt"
para incluir vários (todos) os documentos da pasta no versionamento: "git add ." (sem as aspas)
para enviar os arquivos para o github, ex: git commit -m "commit inicial"
se não tiver cadastrado a conta ainda: git config --global user.email "adenissegall@live.com", aperta enter e depois: git config --global user.name "adenissegall"
quando já tiver setado o usuário do github: git commit -m "commit inicial"
para inserir os arquivos no github: git push
mas vai pedir para especificar o local, então vai no github manualmente e cria um repositorio
pega o link do repositorio e insere o seguinte comando: git remote add origin https://github.com/adenissegall/teste_git
valida com o navegador se for preciso, autorizar
geralmente o arquivo cai na branch master, então ao fazer isso, acessa a "main" e clica em master pra ver se foi
se o arquivo foi alterado, é só dar o comando git status para ver se tem arquivo alterado na pasta
se tiver arquivo alterado:
git add . 
git commit -m "comentario sobre o commit"
git push
para ver as versões criadas: git reflog
para voltar o arquivo para uma versão anterior: git reset --hard + o codigo da versão que apareceu no git reflog. Ex: git reset --hard 5a1cca8
para ver as branchs que existem na conta: git branch
para criar uma branch: git + nome da branch ex: git DEV
para ver se foi criado corretamente: git branch
para mudar de branch: git checkout + nome da branch. Ex: git checkout DEV
para ver se deu certo: git branch Se tiver dado certo, a branch DEV vai ficar verde e com um asterisco do lado
se tiver algum arquivo alterado na pasta e quiser adicionar nessa branch nova, git status, git add . , git commit -m "nome da alteração", git push.
Provavelmente vai dar erro e pedir para inserir esse comando: git push --set-upstream origin DEV (enter)
